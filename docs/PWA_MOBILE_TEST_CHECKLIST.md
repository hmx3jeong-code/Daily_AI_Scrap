# Daily AI Brief PWA 모바일 테스트 체크리스트

테스트 대상 URL:
- `https://hmx3jeong-code.github.io/Daily_AI_Scrap/`

테스트 기준:
- 날짜 기준 최신 데이터 로딩
- 앱 설치 가능
- 오프라인 동작
- 외부 API 호출 동의/비동의 동작

## 1. Android (Chrome) 테스트

1. URL 접속 후 상단 설치 안내/`앱 설치` 버튼 표시 확인
2. `앱 설치` 실행 후 홈 화면 아이콘 생성 확인
3. 홈 화면 아이콘으로 앱 실행 시 브라우저 UI 없이 standalone 실행 확인
4. 앱 첫 화면 메타 정보(Report date/Article count/Generated at) 정상 표시 확인
5. 비행기 모드 전환 후 앱 재실행: 캐시된 최신 리포트 표시 확인
6. 과거 리포트 링크 클릭 동작 확인(오프라인 시 접근 제한은 허용)
7. 질문 기능 기본 OFF 상태 확인
8. 질문 기능 동의 체크 OFF 상태에서 질문 시 차단 메시지 확인
9. 질문 기능 동의 체크 ON + API Key 저장 후 질문 동작 확인
10. API Key 삭제 후 질문 재시도 시 키 미설정 안내 확인

## 2. iOS (Safari) 테스트

1. URL 접속 후 설치 안내 문구(공유 버튼 > 홈 화면에 추가) 표시 확인
2. Safari 공유 메뉴에서 `홈 화면에 추가` 수행
3. 홈 화면 아이콘 실행 시 standalone 스타일로 열리는지 확인
4. 기본 데이터 로딩 정상 확인
5. 네트워크 OFF 후 재실행 시 캐시된 리포트 또는 오프라인 페이지 확인
6. 설치 후 재방문 시 설치 버튼이 숨김 상태인지 확인
7. 질문 기능 동의/비동의 동작이 Android와 동일한지 확인

## 3. 보안/안정성 점검

1. 기사 링크 클릭 시 새 탭 열림 + `window.opener` 차단(`noopener`) 확인
2. 비정상 URL(`javascript:` 등)이 렌더링되지 않는지 확인
3. 콘솔 에러 확인:
   - `manifest` 로딩 실패 없음
   - `service worker` 등록 실패 없음
4. 새 배포 후 강력 새로고침으로 최신 SW 반영 확인

## 4. 장애 대응 체크

1. `docs/data/latest.json` 없거나 404인 상황에서 오류 메시지 노출 확인
2. 네트워크 타임아웃 상황에서 캐시 fallback 동작 확인
3. 캐시 초기화 후 첫 진입 시 재다운로드/재렌더 정상 확인

## 5. 릴리즈 전 최종 확인

1. GitHub Actions로 `docs/data/latest.json` 갱신 후 사이트 반영 확인
2. 모바일 2종(Android/iOS) 캡처 이미지 확보
3. 사용자 공지 문구:
   - "모바일 홈 화면에 설치 가능"
   - "질문 기능은 기본 OFF, 동의 후 사용"
