- [Codex Security가 SAST 보고서를 포함하지 않는 이유](https://openai.com/index/why-codex-security-doesnt-include-sast) | OpenAI News | 2026-03-16 09:00
  - 요약: OpenAI는 Codex Security가 기존의 정적 애플리케이션 보안 테스트(SAST) 방식을 채택하지 않는 이유를 상세히 설명했다. 전통적인 SAST 도구는 코드의 패턴을 기반으로 취약점을 탐색하여 오탐지율이 높다는 한계가 있다. 반면 Codex Security는 AI 기반의 제약 조건 추론과 검증 기술을 활용한다. 이 방식은 코드의 문맥을 깊이 있게 이해하여 실제 존재하는 취약점만을 정확히 식별한다. 결과적으로 개발자는 불필요한 경고를 확인하는 시간을 줄이고 핵심적인 보안 문제 해결에 집중할 수 있다. 이러한 접근은 단순한 패턴 매칭을 넘어 코드의 논리적 흐름을 분석하는 데 중점을 둔다. AI 모델은 코드의 의도와 제약 조건을 추론하여 보안 위협을 실시간으로 평가한다. 이는 기존 도구들이 놓치기 쉬운 복잡한 보안 결함을 찾아내는 데 효과적이다. 실무적으로는 보안 검토 과정의 효율성을 극대화하고 개발 생산성을 높이는 데 기여한다. OpenAI는 이러한 기술적 전환이 보안 자동화의 새로운 표준이 될 것으로 기대하고 있다. 결과적으로 Codex Security는 더 적은 오탐지로 신뢰도 높은 보안 분석 결과를 제공한다.
  - 키워드: Codex Security, SAST, AI 보안, 취약점 분석, 제약 조건 추론

### 멀티모달 (1)

  - 요약: 애플 머신러닝 연구팀은 밀집 이미지 캡셔닝의 효율성을 높이기 위해 'RubiCap' 프레임워크를 발표했다. 밀집 이미지 캡셔닝은 비전-언어 사전 학습과 텍스트-이미지 생성의 핵심 기술이지만, 전문가 수준의 주석을 대규모로 확보하는 데는 막대한 비용이 발생한다. 기존의 강력한 비전-언어 모델(VLM)을 활용한 합성 캡셔닝은 대안으로 사용되지만, 지도 증류 방식은 출력의 다양성이 부족하고 일반화 성능이 낮다는 한계가 있다. 연구팀은 이러한 문제를 해결하기 위해 강화학습(RL)을 도입했다. 기존 강화학습은 결정론적 검사기가 존재하는 영역에서 주로 성공했으나, 캡셔닝은 개방형 작업이라 적용이 어려웠다. RubiCap은 루브릭(Rubric)을 가이드로 활용하여 이러한 제약을 극복했다. 이 모델은 루브릭 기반의 보상 체계를 통해 모델이 생성하는 캡션의 품질과 다양성을 동시에 개선한다. 이번 연구는 고품질 데이터 확보가 어려운 상황에서 모델의 학습 효율을 극대화하는 실무적 대안을 제시한다. 특히 복잡한 이미지 내의 다양한 객체를 정확하게 묘사해야 하는 작업에서 성능 향상을 기대할 수 있다. 향후 이 기술은 자동화된 데이터 생성 파이프라인의 신뢰성을 높이는 데 기여할 것으로 보인다. 결과적으로 RubiCap은 데이터 주석 비용을 절감하면서도 모델의 일반화 능력을 강화하는 효과적인 방법론을 입증했다.
  - 키워드: RubiCap, 밀집 이미지 캡셔닝, 강화학습, 비전-언어 모델, 데이터 증류
